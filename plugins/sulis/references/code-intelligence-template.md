# CODE_INTELLIGENCE.md Template

> **Status:** Active — v0.7.0

The canonical schema for `.architecture/{project}/CODE_INTELLIGENCE.md`,
produced by `/sulis:analyse-codebase`. SEA's blueprint, decompose, harden, and verify
skills parse this template — do not deviate from heading structure.

Every section entry is marked with **provenance**:

- `[deterministic: ast-grep]` — extracted programmatically by ast-grep
- `[deterministic: lizard]` — measured by lizard
- `[deterministic: scc]` — measured by scc
- `[llm-inferred, confidence: low|medium|high]` — Phase 2 synthesis

## How Downstream Plugins Read This

1. **Parse the frontmatter** — `Generated`, `Toolchain`, `Source mtimes`.
2. **Honour the refresh rule** — if any tracked source mtime is newer than
   the probe's `Generated`, recommend re-running `/sulis:analyse-codebase` before
   proceeding.
3. **Cite by section** — when blueprint says "Reuse `result.ts`", it cites
   `CODE_INTELLIGENCE.md § Reusable Abstraction Catalogue`. When decompose
   names a target file, it cites Capability Inventory.
4. **Respect Wrapper Rot list** — verify's P6 cross-checks proposed Wrap WPs
   against this list. Decompose's Wrap Audit checks here.
5. **Treat Recommendations as advisory** — not as gates. The user or
   blueprint decides whether each recommendation lands as a WP.
