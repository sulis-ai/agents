## Cumulative history (through v0.2.0)

> Migrated from `plugin.json` description, 2026-05-23.

Founder Concierge — the single entry point for non-technical founders building a product through the Sulis AI marketplace. Orchestrates the journey from idea → requirements → design → implementation → verification → security in plain English. Spawns specialist agents (SRD, SEA, sulis-security, sulis-context, sulis-execution) at the right time, reads their produced artifacts, translates technical output into founder English, maintains journey state across sessions in .concierge/{project}/JOURNEY.md. Applies AAF-01..AAF-09 aggressively at every founder-facing message. Start a session with `claude --agent sulis-concierge`. v0.1.6: cites Founder English (FE-01..FE-10) at plugins/srd/references/founder-english.md — every founder-facing chat message AND founder-readable artifact write passes the FE-06 five-point check (strip internal IDs, translate filenames, expand acronyms, strip internal taxonomy, read-aloud test). Lead with outcomes; concrete over abstract; no mechanism narration. v0.1.7: deep restructure adopting patterns from tria's explorer.yaml + production-guardian.md + retired concierge.yaml + strategy.yaml. Adds: 'you think this internally / you say this externally' framing; a five-phase pre-emission mechanical gate (Load → Categorise → Triage → Decide → Emit); 5-lens analysis for specialist output (gap-matching / strategy-divergence / journey-alignment / feature-implications / opportunity-detection); the 'What I heard / What I noticed / What I recommend' synthesis template; explicit three-state output model (PROCEED / GATHER / BLOCKED) with enumerated forbidden output shapes (three-questions, specialist-output tables, mechanism narration, menus of next steps, permission-theatre closures). Closes the relay-and-translate failure that v1.15.0's FE block didn't catch (anchor case 3 in founder-english.md). v0.1.8: closes the wording-variant gap from v1.15.2 partial fix. Expands the permission-theatre wording list with pattern-match anchors ("Want me to invoke X on Y path?", "Should I walk those next?", "Would you prefer A or B?", "Let me know if...", "Happy to..."). Adds Recommendation = Action MUST rule: when you make a recommendation, you act on it in the same message; the closing line is action-then-report, never a permission question. Catches the failure pattern where the agent does the analysis, makes the right call, then asks the CEO to greenlight the call it just made. v0.1.9: extends the Pre-Emission Gate explicitly to mid-task status updates AND Agent-tool `description` parameters — they are founder-facing chrome and pass the same FE-06 scan as chat messages. Adds BAD/GOOD pairs covering background-task announcements ("spawning the executor for WP-AUTO-012" → "going back to the stuck task"), Agent-tool descriptions ("Retry WP-AUTO-012 after SEA reconciliation" → "Resuming after the architect's fix"), and pipeline status ("WP-AUTO-012 retry executor running in background alongside the two test-WP pipelines" → "three tasks running; I'll come back when any finishes"). Closes the v1.15.3 partial-fix where chat messages translated but the harness chrome did not. v0.2.0: change-aware initiation per the Change Work Standard (CW-01..CW-08 published by SRD v1.22.0). Before invoking any specialist that produces artifacts (SRD, SEA, executor), the concierge picks a change primitive + slug from the founder's intent, runs sulis-change start to provision a dedicated change branch + worktree, and invokes downstream specialists inside that worktree. Active change recorded in JOURNEY.md so subsequent sessions resume in the right workspace. Trivial-change carve-out documented (CW-05): typo fixes / ≤30 line text-only edits skip the change ceremony. Composition: concierge calls sulis-execution v0.12.0+'s sulis-change CLI; specialist invocations stay unchanged except --repo-root points at the change worktree; per-WP executor uses the change branch as base via the --base-branch threading.
