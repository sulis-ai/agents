# Retry policy — next_delay backoff curve

## Why

This is the backoff arithmetic the spec's "persistent" retry budget requires:
exponential backoff + full jitter, capped at a ~10–15-min wall-clock budget,
returning the give-up signal on exhaustion (ADR-002). It is a pure function,
testable with a seeded RNG and no real sleeping (the maintenance-test
precedent; MEA-09). It is the second producer in the WP-001 contract,
parallel to and independent of the classifier (WP-002).

## What changes

- `next_delay(attempt, elapsed_seconds, policy) -> float | None` (home:
  `recovery.py`, beside the `RetryPolicy` value object from WP-001).
- Consumes the `RetryPolicy` value object (WP-001); no other dependency.

## How

Per ADR-002 and the contract stub table:

- Full jitter (AWS convention, CP-01):
  `random_between(0, min(policy.max_delay_seconds, policy.base_delay_seconds * policy.multiplier ** attempt))`.
- Wall-clock budget, not attempt count: return `None` when the next attempt
  would push cumulative `elapsed_seconds` past `policy.total_budget_seconds`.
  `None` is the driver's signal to reclassify `TRANSIENT_BLIP → DEAD_END`.
- RNG is injectable (default `random.random`) so tests seed it; the clock is
  **not** this function's concern — `elapsed_seconds` is passed in (the driver
  owns the injectable clock, WP-005).

## Tests (Red → Green → Blue)

**Red** — `test_session_manager_retry_policy.py` parametrized over the WP-001
`next_delay` stub table, with a seeded RNG; assert the jitter bounds at
attempts 0/3/10 and `None` at budget exhaustion. See it fail (no function).

**Green** — implement the curve until the table passes.

**Blue** — assert no `time.sleep` is reachable from `next_delay` (it is pure
