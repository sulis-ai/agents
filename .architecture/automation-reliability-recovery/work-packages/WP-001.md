# Reliability-layer data contract — RecoveryClass + RetryPolicy

## Why

This is the producer/consumer seam at the heart of the reliability layer
(CONTRACT_FIRST CF-01). The **classifier** (WP-002) produces a `RecoveryClass`
verdict that the **recovery driver** (WP-005) consumes; the **policy**
(WP-003) produces a backoff delay the driver consumes. Pinning the vocabulary
and the value-object shapes here lets the classifier, the policy, the driver,
and the adapter seam (WP-004/006) all be built and tested **in parallel**
against one agreed source — the verdict words are never re-spelled on either
side (CF-11 spirit).

It comes first because every other WP `dependsOn` it.

## What changes

New shared vocabulary, living with the reliability layer in
`plugins/sulis/scripts/_session_manager/` (exact module split is the
implementer's call — `classifier.py` is the natural home for `RecoveryClass`
per ADR-003; `recovery.py` for `RetryPolicy`/`ReauthTicket` per ADR-002/004 —
but only the *shape* and the *importable surface* are contracted here):

- `RecoveryClass` enum: `TRANSIENT_BLIP | DEAD_END | LOGIN_EXPIRED` (ADR-003).
- `RetryPolicy` frozen dataclass + `DEFAULT_RETRY_POLICY` constant (ADR-002).
- `ReauthTicket` value object: the re-login link the notification carries + a
  completion handle (ADR-003/004).
- `_session_manager/__init__.py` re-exports the new public surface so callers
  import from the package (Form §2.3).

It imports the existing `events.py` `ErrorCategory` + code constants; it does
**not** add or redeclare any error code (ADR-003 — `NOT_AUTHORIZED` already
exists).

## How

Follow `contracts/reliability-layer.contract.md` exactly:

- `RetryPolicy` fields and `DEFAULT_RETRY_POLICY` values are the ones in
  ADR-002 (`base=1.0, max=60.0, mult=2.0, jitter="full", budget=720.0`).
- The contract's truth table (12 rows) + the `next_delay` stub table (4 rows)
  are encoded as a shared pytest fixture/parametrize data structure so WP-002
  and WP-003 each assert against it — one source of truth for the examples
  (CF-04 happy + error + empty cases, incl. the unknown-code → dead-end
  fall-through).

## Tests (Red → Green → Blue)
