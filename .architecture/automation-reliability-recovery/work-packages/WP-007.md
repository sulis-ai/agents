# Manager wiring — attach the RecoveryDriver

## Why

The driver (WP-005) and the Claude detection (WP-006) exist but observe
nothing until the composition root wires them. This WP attaches the driver at
the manager's **error-event observation seam** — the same architectural slot
`LifecycleManager` uses for `_on_process_death` (ADR-001) — so a live turn that
ends in an `error` Event is routed to recovery, while the existing core flow is
untouched. This is the highest-blast-radius WP because it touches the live
`manager.py`; it is deliberately the smallest possible additive change.

## What changes

- `_session_manager/manager.py`:
  - Construct the `RecoveryDriver` at the composition root (one line, beside
    the `LifecycleManager` construction), injecting the classifier, the default
    `RetryPolicy`, the clock, and the per-session adapter.
  - Expose / attach the error-observation hook (sibling to `_on_process_death`)
    that hands each `error`-kind Event from a **live** session to the driver,
    and explicitly does **not** hand it process-death `STDIN_BROKEN` errors
    (lifecycle owns those — the no-double-handling rule, TDD §3.1 step 1).

## How

Per ADR-001 and TDD §2.2. The manager already exposes attach seams
(`_on_process_death`, `_maintenance_tick`, `_guard`) "so the wave-4 Armor WPs
attach without editing the core flow" — this is one of those WPs. `EXPAND-Extend`:
a new hook + construction line, additive, the LifecycleManager precedent.

Per EP-07 / TDD §2.2 + §5: the manager change is **additive** (a new hook + a
construction line), not a restructuring of existing behaviour, so it does not
require a characterisation test of its own. The guard against regression is the
**existing** session-manager unit + integration suite staying green — those
tests already characterise turn-complete / one-in-flight / state-machine
behaviour, and this WP must not perturb them.

## Tests (Red → Green → Blue)

**Red** — `test_session_manager_recovery_wiring.py`: drive a live (scripted-child)
session to emit an `error` Event and assert it reaches the driver; drive a
process-death `STDIN_BROKEN` and assert it does **not** reach the driver
(lifecycle path). See it fail (no hook).

**Green** — add the construction line + hook until routing is correct.

**Blue** — run the **full existing** `test_session_manager_core.py` +
`test_session_manager_host.py` (unit + integration) and confirm byte-unchanged
behaviour (the must-not-change acceptance). Confirm the diff to `manager.py` is
purely additive.

## Rollback

Revert the commit. Detaches the driver; manager returns to pre-change behaviour.
