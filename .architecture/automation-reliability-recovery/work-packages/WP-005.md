# RecoveryDriver — the turn-level Armor primitive

## Why

This is the heart of the change: the driver that mirrors `LifecycleManager`'s
around-the-core shape (ADR-001) and turns a stopped turn into retry / abandon /
pause→resume. It is the consumer in the WP-001 contract — it consumes the
classifier verdict (WP-002), the backoff curve (WP-003), and the adapter's
re-auth (WP-004), and drives recovery through the manager's existing public
primitives without touching `Session.submit`, the FIFO, or `turn_complete`.

## What changes

- `_session_manager/recovery.py` — the `RecoveryDriver` (lives with the
  `RetryPolicy` value object from WP-001/003).
- Constructed with injected capabilities (the `LifecycleManager` convention):
  the classifier, the `RetryPolicy` (default `DEFAULT_RETRY_POLICY`), an
  injectable `clock` (default `time.monotonic`), and the manager primitives it
  needs — `send` (re-submit), `log.append` (surface every action), the resume
  trigger (`supports_resume` + `resume_ref`), and the per-session adapter.

## How

Per the TDD §3.1 pipeline (order is load-bearing) and ADR-001/002/004:

1. **Skip process-death.** A `STDIN_BROKEN` "died mid-turn" is `LifecycleManager`'s
   job — the driver no-ops on it (the two seams are siblings, not stacked).
2. **Classify** via `classifier.classify`, passing the session adapter's
   `classify_failure` hint.
3. **Branch:**
   - **transient-blip** → `next_delay(attempt, elapsed, policy)`; if a delay,
     append a "retry scheduled" `error` Event (reusing the observed code),
     wait the jittered delay against the **injected clock**, then re-submit via
     the existing `manager.send` (the FIFO serialises it — a retry is just
     another turn, never a second in-flight). If `None` → fall to dead-end with
     "retry budget exhausted".
   - **dead-end** → append a typed "abandoned: <reason>" `error` Event and stop.
     **Budget not consulted.**
   - **login-expired** → the pause→notify→resume flow (ADR-004): `reauth()` →
     append a `NOT_AUTHORIZED` Event carrying the re-login link → pause (hold
     the session out of the retry loop, do **not** burn the budget) pending the
     ticket → on completion resume via the existing `supports_resume` +
     `resume_ref` path; the agent re-runs the incomplete step (no fabricated
     completion).
4. **Observe.** Every branch appends an Event to the existing log (no new
   stream) — the emitted-Event shapes are the contract's table.

Recovery is per-session/keyed; the driver never touches a session other than
the one whose error it observed (isolation, ADR-001 §3.4).
