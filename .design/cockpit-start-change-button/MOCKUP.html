State 1 — The front door, cold start (no changes yet)

One primary action lives at the top of the sidebar: Start something new. It is the only blue button in the chrome, so it reads as the front door. When the founder has no work yet, the workspace itself invites them in rather than showing an empty list.

State 6 — Starting… (loading) and the error fallback

After “Start this work”, the workspace is created — a brief, honest progress moment. If it can't start (e.g. a branch name clash or the engine is busy), the founder sees a plain-language error with a single retry, never a dead end.

After start — hand-off into the existing change workspace
