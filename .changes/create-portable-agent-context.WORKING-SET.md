## 1. Problem  (→ Opportunity)
Portable, provider-agnostic context payload: rich curated context injected by default + raw full message/thread record tracked and discoverable on demand (progressive disclosure). Assembled from the durable sources Sulis already owns (Working Set, brain, brief/CONTEXT) + a per-session/thread message log. Enables headless failover + hardened resume/audit.

## 2. Current best solution  (→ Design)
TDD + 5 ADRs + 8 WPs at `.architecture/portable-agent-context/` (tier M).
Three new pieces over existing seams (no re-architecture): (1) a durable,
append-only ThreadStore; (2) a ContextPayloadAssembler (generated view); (3) a
discovery seam = inline payload pointer (rich-by-default) + a read-only
`thread_context` MCP tool (raw-on-demand). All fed by the existing
provider-neutral `events.Event` vocab + the brief-injection seam. Shape conforms
field-for-field to the platform thread-sdk model. CENTRAL RULING (ADR-002):
HYBRID — local-first store that conforms to the platform contract, syncable
later behind the same transport-agnostic contract (rejected: calling the hosted
service = hard dep on parked/unreachable infra; pure-local = forks the model).
